# UI — уведомления, диалоги, меню

Reunion даёт доступ к трём видам UI: всплывающие уведомления (Bulletins), диалоги (AlertDialog), и пункты контекстных меню. Все они — это элементы самого приложения, выглядят нативно.

---

## BulletinHelper — всплывающие уведомления

Bulletins — полоска которая появляется снизу экрана на несколько секунд. Используется для коротких информационных сообщений.

```
import ui.bulletin(BulletinHelper)
import client_utils(get_last_fragment)
```

`get_last_fragment()` — возвращает текущий активный экран приложения. Большинство UI-методов требуют его как аргумент. Можно не передавать — тогда библиотека попробует найти его сама, но лучше передавать явно.

### Стандартные уведомления

```
val fragment = get_last_fragment()

BulletinHelper.show_info("Информационное сообщение", fragment)
BulletinHelper.show_error("Что-то пошло не так", fragment)
BulletinHelper.show_success("Готово!", fragment)
```

### Предустановленные контекстные

```
BulletinHelper.show_copied_to_clipboard("Текст скопирован", fragment)
BulletinHelper.show_link_copied(false, fragment)          // false = публичная ссылка
BulletinHelper.show_file_saved_to_gallery(false, 1, fragment)    // false = фото, 1 = количество
BulletinHelper.show_file_saved_to_downloads("MUSIC", 1, fragment)
```

### Кастомные уведомления с Lottie-анимацией

```
import org.telegram.messenger(R as R_tg)

// однострочное с анимацией
BulletinHelper.show_simple("Processing...", R_tg.raw.timer, fragment)

// двухстрочное
BulletinHelper.show_two_line("Готово", "Файл сохранён в галерею", R_tg.raw.ic_download_done, fragment)
```

`R_tg.raw.*` — ссылки на Lottie-анимации внутри приложения. Несколько часто используемых:

| Имя | Когда использовать |
|---|---|
| `R_tg.raw.info` | Информация |
| `R_tg.raw.timer` | Ожидание, обработка |
| `R_tg.raw.success` | Успех |
| `R_tg.raw.delete` | Удаление |
| `R_tg.raw.ic_download_done` | Загрузка завершена |

Это не полный список. Все Lottie-анимации хранятся в ресурсах Telegram как `.json` файлы — найти их можно в исходном коде по пути `TMessagesProj/src/main/res/raw/`. Например `info.json` доступна как `R_tg.raw.info`, `success.json` — как `R_tg.raw.success`. Полный список можно посмотреть в репозитории [TelegramAndroid](https://github.com/DrKLO/Telegram) в этой папке.

### Уведомление с кнопкой

```
BulletinHelper.show_with_button(
    "Настройки обновлены",
    R_tg.raw.info,
    "Открыть",
    () -> log("button clicked"),
    fragment
)
```

### Уведомление с отменой (Undo)

```
BulletinHelper.show_undo(
    "Сообщение удалено",
    on_undo: () -> log("undone"),
    on_action: () -> log("permanently deleted"),
    fragment: fragment
)
```

## AlertDialogBuilder — диалоги

Диалоги отображаются поверх экрана и блокируют взаимодействие пока пользователь не ответит.

```
import ui.alert(AlertDialogBuilder)
import client_utils(get_last_fragment)
```

Для создания диалога нужен контекст Activity:

```
val fragment = get_last_fragment()
val activity = fragment.getParentActivity()
```

### Простой диалог с кнопками

```
val dialog = AlertDialogBuilder(activity)
dialog.set_title("Предупреждение")
dialog.set_message("Вы уверены что хотите продолжить?")
dialog.set_positive_button("Да", (bld, _) -> bld.dismiss())
dialog.set_negative_button("Отмена", (bld, _) -> bld.dismiss())
dialog.show()
```

### Диалог со списком

```
val items = ["Опция А", "Опция Б", "Опция В"]

val dialog = AlertDialogBuilder(activity)
dialog.set_title("Выберите")
dialog.set_items(items, (bld, idx) ->
    log("Selected: ${items[idx]}")
    bld.dismiss()
)
dialog.set_negative_button("Отмена", (bld, _) -> bld.dismiss())
dialog.show()
```

### Диалог загрузки (спиннер)

```
val loading = AlertDialogBuilder(activity, AlertDialogBuilder.ALERT_TYPE_SPINNER)
loading.set_title("Загрузка...")
loading.set_message("Пожалуйста подождите")
loading.set_cancelable(false)
loading.show()

// потом когда загрузка завершится:
loading.dismiss()
```

`set_cancelable(false)` — пользователь не может закрыть диалог кнопкой назад или тапом за пределами.

## Меню — `menu_item`

Позволяет добавить свои пункты в контекстное меню сообщения или в меню профиля. Пункты регистрируются в `on_load` и удаляются автоматически при выгрузке плагина.

```
import base_plugin(BasePlugin, MenuItemData, MenuItemType)

plugin MyPlugin {
    fun on_load() {
        menu_item {
            type: MESSAGE_CONTEXT_MENU
            text: "Скопировать ID"
            icon: "msg_copy"
            on_click: (context) -> log("clicked")
        }

        menu_item {
            type: PROFILE_ACTION_MENU
            text: "Показать информацию"
            icon: "msg_info"
            on_click: (context) -> log("profile clicked")
        }
    }
}
```

### Типы меню

| Тип | Где появляется |
|---|---|
| `MESSAGE_CONTEXT_MENU` | Контекстное меню при долгом нажатии на сообщение |
| `PROFILE_ACTION_MENU` | Меню действий на странице профиля |
| `DRAWER_MENU` | Боковое меню приложения |

### Объект `context` в колбэке

`context` — словарь с данными о том на что нажали:

```
menu_item {
    type: MESSAGE_CONTEXT_MENU
    text: "Log Message"
    icon: "msg_info"
    on_click: (context) -> {
        val message = context.get("message")
        if message {
            log("Message ID: ${message.getId()}")
            log("Text: ${message.messageText}")
            log("From: ${message.getSenderId()}")
        }
    }
}
```

Для `PROFILE_ACTION_MENU`:

```
on_click: (context) -> {
    val user = context.get("user")
    if user {
        log("User: ${user.first_name} (ID: ${user.id})")
    }
}
```

## Все вызовы на UI-потоке

Все методы BulletinHelper, AlertDialogBuilder и menu_item **должны вызываться на UI-потоке**. Если ты делаешь что-то в фоне через `run_on_queue` — перед вызовом UI вернись на главный поток:

```
import android_utils(run_on_ui_thread)
import client_utils(run_on_queue, get_last_fragment)

run_on_queue(() ->
    val result = doSomethingHeavy()
    run_on_ui_thread(() ->
        val fragment = get_last_fragment()
        BulletinHelper.show_success("Готово: $result", fragment)
    )
)
```

Если вызвать UI-метод из фонового потока — приложение упадёт с исключением.
